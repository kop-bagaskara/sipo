## 1. PANDUAN PENGGUNAAN UNTUK PESERTA TRAINING

### 1.1 Akses dan Login ke Sistem

**Langkah-langkah Login:**

1. Buka browser web (Google Chrome, Mozilla Firefox, atau Microsoft Edge)
2. Ketikkan URL sistem training pada address bar
3. Halaman login akan ditampilkan
4. Masukkan **Username** dan **Password** akun Anda
5. Klik tombol **Login**


6. Setelah berhasil login, Anda akan diarahkan ke Dashboard utama


---

### 1.2 Melihat Daftar Training Tersedia

**Langkah-langkah Melihat Daftar Training:**

1. Dari menu navigasi di sebelah kiri/atas, klik menu **Portal Training** atau **Training**
2. Halaman daftar training akan ditampilkan
3. Anda dapat melihat daftar training yang telah ditugaskan kepada Anda


**Informasi yang Ditampilkan:**

| Kolom | Keterangan |
|-------|-------------|
| Judul Training | Nama/judul training |
| Tanggal | Periode pelaksanaan training |
| Status | Status training (Belum Dimulai, Sedang Berlangsung, Selesai, Telat) |
| Progress | Persentase penyelesaian training |
| Aksi | Tombol untuk melihat detail atau mengikuti training |

---

### 1.3 Mengikuti Training

**Langkah-langkah Mengikuti Training:**

1. Klik tombol **Detail** atau **Lihat** pada training yang akan diikuti
2. Halaman detail training akan ditampilkan


**Informasi Detail Training:**
- **Kode Training**: Kode unik identifikasi training
- **Judul**: Nama lengkap training
- **Deskripsi**: Penjelasan mengenai tujuan dan materi training
- **Tanggal Mulai & Selesai**: Periode pelaksanaan
- **Tipe**: Tipe training (Online/Offline/Hybrid)
- **Kuota**: Jumlah maksimal peserta (jika ada)
- **Status Penyelesaian**: Progress Anda dalam training ini

---

### 1.4 Mengakses dan Mempelajari Materi Training

**Langkah-langkah Mengakses Materi:**

1. Pada halaman detail training, scroll ke bagian **Materi Training**
2. Daftar materi yang tersedia akan ditampilkan


3. Klik pada judul materi yang ingin dipelajari
4. Materi akan terbuka dalam tampilan pembaca


5. Baca dan pelajari materi secara seksama
6. Setelah selesai, Anda dapat kembali ke daftar materi untuk melanjutkan ke materi berikutnya

**Tips Mempelajari Materi:**
- Pelajari materi secara berurutan untuk pemahaman yang lebih baik
- Catat poin-poin penting dari setiap materi
- Ulangi membaca materi jika diperlukan
- Pastikan koneksi internet stabil saat mengakses materi

---

### 1.5 Mengerjakan Kuis/Ujian

**Ketentuan Pengerjaan Ujian:**

Perhatikan ketentuan berikut sebelum mengerjakan ujian:
- Pastikan Anda telah mempelajari semua materi terlebih dahulu
- Beberapa ujian memiliki batas waktu pengerjaan
- Beberapa ujian mungkin tidak dapat diulang
- Pastikan koneksi internet stabil

**Langkah-langkah Mengerjakan Ujian:**

1. Pada halaman detail training, scroll ke bagian **Ujian/Kuis**
2. Perhatikan informasi ujian yang ditampilkan:
   - Jumlah soal
   - Waktu pengerjaan (jika ada)
   - Nilai minimum kelulusan
   - Apakah ujian dapat diulang


3. Jika sudah siap, klik tombol **Mulai Ujian** atau **Start Exam**
4. Konfirmasi untuk memulai ujian


5. Halaman ujian akan ditampilkan dengan soal pertama


6. Baca pertanyaan dengan seksama
7. Pilih jawaban yang menurut Anda benar dengan mengklik pilihan jawaban
8. Klik tombol **Next** atau **Selanjutnya** untuk soal berikutnya
9. Ulangi langkah tersebut hingga semua soal terjawab

**Selama Pengerjaan Ujian:**
- Perhatikan **sisa waktu** (jika ada timer)
- Beberapa sistem memperbolehkan kembali ke soal sebelumnya
- Beberapa sistem menerapkan sistem tidak boleh kembali
- Jangan menutup browser selama pengerjaan

10. Setelah semua soal terjawab, klik tombol **Submit** atau **Kirim Jawaban**
11. Konfirmasi pengiriman jawaban


12. Hasil ujian akan ditampilkan


---

### 1.6 Melihat Riwayat dan Sertifikat

**Melihat Riwayat Training:**

1. Klik menu **Riwayat Training** atau **History**
2. Daftar training yang pernah Anda ikuti akan ditampilkan


**Informasi Riwayat:**
- Nama training
- Tanggal penyelesaian
- Nilai ujian (jika ada)
- Status kelulusan
- Link sertifikat (jika lulus)

**Mengunduh Sertifikat:**

1. Pada riwayat training, klik tombol **Sertifikat** pada training yang Anda lulus
2. Sertifikat akan terbuka dalam format PDF


3. Klik tombol download atau simpan sertifikat
4. Anda dapat mencetak sertifikat untuk keperluan administrasi

## 2. PANDUAN PENGGUNAAN UNTUK KOORDINATOR TRAINING/HR

### 2.1 Dashboard Koordinator Training

**Mengakses Dashboard:**

1. Login ke sistem dengan akun Koordinator/HR
2. Klik menu **Training Management** atau **Master Training** di sidebar
3. Dashboard koordinator akan ditampilkan


**Informasi pada Dashboard:**
- Total training aktif
- Total peserta training
- Training yang akan datang
- Training sedang berlangsung
- Statistik kelulusan
- Notifikasi training yang memerlukan perhatian

---

### 2.2 Membuat Training Master Baru

**Langkah-langkah Membuat Training:**

1. Klik menu **Training Masters** → **Tambah Training** atau **Create Training**
2. Form pembuatan training akan ditampilkan


3. Isi informasi training:

   **Bagian 1: Informasi Dasar**
   - **Kode Training**: Masukkan kode unik (contoh: TRN-001, MSD-2025-001)
   - **Judul Training**: Masukkan nama lengkap training
   - **Deskripsi**: Jelaskan tujuan, materi, dan target peserta
   - **Kategori**: Pilih kategori (Soft Skill, Technical Skill, Leadership, dll)
   - **Tipe Training**: Pilih tipe (Online, Offline, Hybrid)

   **Bagian 2: Schedule**
   - **Tanggal Mulai**: Tentukan tanggal dimulainya training
   - **Tanggal Selesai**: Tentukan tanggal berakhirnya training
   - **Durasi**: Estimasi durasi penyelesaisan (jam)

   **Bagian 3: Pengaturan**
   - **Kuota Peserta**: Tentukan jumlah maksimal peserta (kosongkan jika tidak terbatas)
   - **Allow Retry**: Centang jika peserta diperbolehkan mengulang ujian
   - **Nilai Minimum**: Tentukan nilai minimum untuk kelulusan
   - **Status**: Set status training (Draft, Published, Archived)


4. Klik tombol **Simpan** atau **Save** untuk menyimpan
5. Training master berhasil dibuat


---

### 2.3 Mengupload Materi Training

**Langkah-langkah Mengupload Materi:**

1. Dari daftar Training Masters, klik tombol **Edit** pada training yang akan ditambah materi
2. Scroll ke bagian **Materi Training**
3. Klik tombol **Tambah Materi** atau **Upload Material**


4. Form upload materi akan ditampilkan


5. Isi informasi materi:
   - **Judul Materi**: Masukkan judul untuk materi ini
   - **Deskripsi**: Jelaskan secara singkat isi materi
   - **File**: Klik tombol **Browse** atau **Choose File** untuk memilih file dari komputer
   - **Tipe File**: Pilih tipe materi (Document, Presentation, Video, Link)
   - **Urutan**: Tentukan urutan materi (1, 2, 3, dst)

6. Tipe file yang didukung:
   - **Document**: PDF, DOC, DOCX
   - **Presentation**: PPT, PPTX
   - **Video**: MP4, AVI, MKV
   - **Link**: URL eksternal (YouTube, dll)

7. Klik tombol **Upload** atau **Simpan**
8. Materi berhasil ditambahkan ke dalam training


9. Ulangi langkah untuk menambahkan beberapa materi

---

### 2.4 Membuat Bank Soal dan Kuis

**Langkah-langkah Membuat Bank Soal:**

1. Klik menu **Question Banks** atau **Bank Soal**
2. Klik tombol **Tambah Bank Soal** atau **Create Question Bank**


3. Form pembuatan bank soal akan ditampilkan


4. Isi informasi bank soal:
   - **Nama Bank Soal**: Masukkan nama untuk kumpulan soal ini
   - **Kategori**: Pilih kategori soal
   - **Tipe Soal**: Pilih tipe (Pilihan Ganda, Essay, Benar/Salah)
   - **Deskripsi**: Jelaskan secara singkat

5. Klik **Simpan** untuk membuat bank soal

---

### 2.5 Menambahkan Soal ke Bank Soal

**Langkah-langkah Menambahkan Soal:**

1. Dari daftar Bank Soal, klik **Edit** atau **Manage Questions** pada bank soal yang dipilih
2. Halaman manajemen soal akan ditampilkan
3. Klik tombol **Tambah Soal** atau **Add Question**


4. Form tambah soal akan ditampilkan


5. Isi informasi soal:
   - **Pertanyaan**: Masukkan teks pertanyaan
   - **Tipe Pertanyaan**: Pilihan ganda (A, B, C, D, E)
   - **Pilihan Jawaban**: Masukkan semua pilihan jawaban yang tersedia
   - **Kunci Jawaban**: Pilih atau tandai jawaban yang benar
   - **Nilai/Poin**: Tentukan nilai untuk soal ini
   - **Urutan**: Tentukan urutan penampilan soal

6. Klik **Simpan** untuk menambahkan soal


7. Ulangi langkah untuk menambahkan soal lainnya

**Tips Pembuatan Soal:**
- Buat pertanyaan yang jelas dan tidak ambigu
- Sediakan pilihan jawaban yang masuk akal
- Sebar tingkat kesulitan soal (mudah, sedang, sulit)
- Review soal sebelum dipublikasikan

---

### 2.6 Menugaskan Training ke Karyawan

**Langkah-langkah Penugasan Training:**

1. Klik menu **Training Assignments** atau **Penugasan Training**
2. Klik tombol **Tambah Penugasan** atau **Create Assignment**


3. Form penugasan training akan ditampilkan


4. Isi informasi penugasan:

   **Bagian 1: Pilih Training**
   - **Training Master**: Pilih training yang akan ditugaskan dari dropdown
   - Informasi training akan ditampilkan secara otomatis

   **Bagian 2: Pilih Peserta**
   Terdapat beberapa cara memilih peserta:

   *Opsi A: Berdasarkan Department*
   - Pilih **Department**
   - Semua karyawan di department tersebut akan terpilih
   - Anda dapat menghapus individual jika perlu

   *Opsi B: Berdasarkan Jabatan*
   - Pilih **Jabatan/Position**
   - Semua karyawan dengan jabatan tersebut akan terpilih

   *Opsi C: Individual*
   - Cari nama karyawan di search box
   - Pilih karyawan satu per satu


   **Bagian 3: Schedule**
   - **Tanggal Penugasan**: Tanggal training ditugaskan (default: hari ini)
   - **Deadline Penyelesaian**: Tentukan batas waktu penyelesaian
   - **Tanggal Mulai Akses**: Tentukan kapan peserta dapat mulai mengakses
   - **Tanggal Selesai Akses**: Tentukan kapan akses ditutup

5. Review daftar peserta yang terpilih


6. Klik **Simpan** untuk membuat penugasan
7. Sistem akan mengirim notifikasi ke semua peserta


---

### 2.7 Memantau Progress Peserta

**Langkah-langkah Monitoring:**

1. Klik menu **Training Assignments**
2. Klik tombol **View** atau **Lihat** pada penugasan yang ingin dipantau
3. Detail progress peserta akan ditampilkan


**Informasi yang Ditampilkan:**
- Daftar lengkap peserta
- Status penyelesaian materi (Belum Mulai, Sedang Proses, Selesai)
- Status pengerjaan ujian (Belum, Sedang, Selesai)
- Nilai ujian (jika sudah dikerjakan)
- Status kelulusan (Lulus/Tidak Lulus)
- Tanggal penyelesaian


**Tindakan yang Dapat Dilakukan:**

1. **Kirim Reminder**
   - Pilih peserta yang ingin diingatkan
   - Klik tombol **Kirim Reminder**
   - Peserta akan menerima notifikasi email


2. **Perpanjang Deadline**
   - Klik tombol **Extend Deadline** pada peserta tertentu
   - Tentukan deadline baru
   - Klik **Simpan**

3. **Reset Ujian**
   - Jika fitur allow retry aktif, Anda dapat mereset ujian peserta
   - Klik tombol **Reset Exam** pada peserta
   - Konfirmasi reset
   - Peserta dapat mengerjakan ujian kembali

4. **Hapus Penugasan**
   - Klik tombol **Hapus** pada peserta
   - Konfirmasi penghapusan
   - Peserta akan dihapus dari penugasan training

---

### 2.8 Mengelola Laporan dan Statistik

**Mengakses Laporan:**

1. Klik menu **Training Reports** atau **Laporan Training**
2. Halaman laporan akan ditampilkan


**Jenis Laporan yang Tersedia:**

**1. Laporan Progress**
- Menunjukkan jumlah peserta dalam setiap status
- Progress completion per training
- Peserta yang belum mulai, sedang proses, dan selesai


**2. Laporan Kelulusan**
- Statistik kelulusan per training
- Persentase kelulusan
- Distribusi nilai


**3. Laporan Nilai**
- Daftar nilai seluruh peserta
- Nilai tertinggi dan terendah
- Rata-rata nilai


**Export Laporan:**
- Klik tombol **Export** atau **Download**
- Pilih format (Excel, PDF, CSV)
- File akan terunduh


---

### 2.9 Mengedit dan Menghapus Training

**Mengedit Training:**

1. Dari daftar Training Masters, klik **Edit** pada training yang akan diubah
2. Ubah informasi yang diperlukan
3. Klik **Update** untuk menyimpan perubahan


**Menghapus Training:**
⚠️ **Perhatian**: Training yang sudah memiliki peserta aktif tidak dapat dihapus

1. Pastikan training tidak memiliki peserta aktif
2. Klik tombol **Hapus** atau **Delete** pada training
3. Konfirmasi penghapusan

### 3.2 Melihat Detail dan Persiapan Training

### 3.3 Membuka Session Training

**Langkah-langkah Membuka Session:**

1. Pada hari dan jam pelaksanaan training, buka detail training
2. Klik tombol **Buka Session** atau **Open Session**


3. Dialog buka session akan ditampilkan


4. Isi informasi session:
   - **Tanggal**: Tanggal pelaksanaan (default: hari ini)
   - **Waktu Mulai**: Jam dimulai
   - **Waktu Selesai**: Perkiraan jam selesai
   - **Lokasi/Platform**: Lokasi offline atau link online (jika applicable)
   - **Catatan**: Informasi tambahan untuk peserta

5. Klik **Buka Session**
6. Sistem akan membuat session baru dan generate **Session Code** unik


7. Catat dan share session code ini ke peserta

---

### 3.4 Memonitor Kehadiran Peserta

**Mekanisme Kehadiran:**

Peserta dapat check-in dengan dua cara:

**Opsi 1: Self Check-in oleh Peserta**
1. Peserta memasukkan session code di sistem mereka
2. Status kehadiran otomatis berubah menjadi "Hadir"

**Opsi 2: Manual Check-in oleh Trainer**
1. Dari halaman session, klik menu **Peserta**
2. Daftar peserta akan ditampilkan
3. Klik tombol **Check-in** pada peserta yang hadir
4. Status berubah menjadi "Hadir"


**Informasi Kehadiran:**
- Nama peserta
- Status kehadiran (Belum Hadir, Hadir, Tidak Hadir)
- Waktu check-in
- Progress training
- Nilai ujian (jika sudah)

---

### 3.5 Memfasilitasi Training

**Selama Training Berlangsung:**

1. **Panduan Akses Materi**
   - Pastikan semua peserta dapat mengakses materi
   - Bantu peserta yang mengalami kesulitan teknis
   - Jelaskan struktur dan flow training

2. **Monitoring Progress**
   - Pantau progress peserta secara berkala dari dashboard
   - Identifikasi peserta yang tertinggal
   - Berikan bantuan tambahan jika diperlukan


3. **Interactive Sessions**
   - Buat session diskusi dan Q&A
   - Berikan study case untuk dikerjakan
   - Facilitasi role play jika applicable untuk materi
   - Gunakan whiteboard atau screen share untuk menjelaskan

4. **Time Management**
   - Pastikan training sesuai dengan schedule
   - Berikan break time yang cukup (biasanya setiap 1-2 jam)
   - Informasikan sisa waktu ke peserta

---

**Langkah-langkah Membuka Ujian:**

1. Pastikan semua materi telah selesai dibahas

4. Dialog konfirmasi akan ditampilkan
5. Berikan instruksi verbal ke peserta:

   - Jumlah soal

   - Kriteria kelulusan

8. Monitor peserta yang sedang mengerjakan:
   - Daftar peserta yang sudah mulai mengerjakan
   - Status pengerjaan (Belum Mulai, Sedang Mengerjakan, Selesai)
   - Sisa waktu (jika ada timer)


---

### 4.2 Instalasi dan Setup

**Persyaratan Sistem (Server Requirements):**

- PHP >= 8.1
- Composer
- MySQL/MariaDB >= 5.7
- Web Server (Apache/Nginx)
- Modul PHP: mbstring, openssl, pdo, tokenizer, xml, ctype, json, bcmath

**Langkah-langkah Instalasi:**

1. **Clone Repository**
   ```bash
   git clone [repository-url]
   cd sipo_krisan
   ```

2. **Install Dependencies**
   ```bash
   composer install
   npm install
   ```

3. **Setup Environment**
   ```bash
   cp .env.example .env
   php artisan key:generate
   ```

4. **Configure Database**

   Edit file `.env`:
   ```env
   DB_DATABASE=sipo_krisan_db
   DB_USERNAME=root
   DB_PASSWORD=your_password
   ```


5. **Run Migrations**
   ```bash
   php artisan migrate
   ```

6. **Run Seeders (Optional)**
   ```bash
   php artisan db:seed
   ```

7. **Create Symlink untuk Storage**
   ```bash
   php artisan storage:link
   ```

8. **Clear Cache**
   ```bash
   php artisan config:clear
   php artisan cache:clear
   php artisan view:clear
   ```

9. **Setup Permissions**
   ```bash
   chmod -R 755 storage
   chmod -R 755 bootstrap/cache
   ```

### 4.3 Manajemen User dan Role

**Membuat User Baru:**

1. Buka terminal di server
2. Jalankan artisan tinker:
   ```bash
   php artisan tinker
   ```

3. Buat user baru:
   ```php
   $user = new \App\Models\User();
   $user->name = 'Nama User';
   $user->email = 'email@example.com';
   $user->password = bcrypt('password');
   $user->role = 'hr'; // atau 'trainer', 'employee'
   $user->save();
   ```

**Role yang Tersedia:**

| Role | Keterangan | Akses |

### 4.4 Manajemen Database

**Backup Database:**

1. **Backup Manual Menggunakan MySQL Dump**
   ```bash
   mysqldump -u username -p database_name > backup_$(date +%Y%m%d).sql
   ```

2. **Backup Otomatis dengan Scheduler**
   - Buat cron job di server:
   ```bash
   0 2 * * * mysqldump -u username -ppassword database_name > /backups/db_$(date +\%Y\%m\%d).sql
   ```

```bash
mysql -u username -p database_name < backup_file.sql
```

### 4.8 Security Best Practices

**1. Environment Configuration:**
- Jangan commit file `.env` ke version control
- Set `APP_DEBUG=false` di production
- Gunakan strong password untuk database
- Gunakan HTTPS (SSL certificate)

**2. File Permissions:**
```bash
# Directories should be 755
find . -type d -exec chmod 755 {} \;

# Files should be 644
find . -type f -exec chmod 644 {} \;

# Storage and cache should be writable
chmod -R 775 storage bootstrap/cache
```

**3. Firewall dan Akses:**
- Batasi akses database hanya dari localhost
- Setup firewall untuk memblokir port yang tidak perlu
- Gunakan fail2ban untuk mencegah brute force attack

**4. Update Berkala:**
```bash
# Update composer dependencies
composer update

# Check vulnerabilities
composer audit

# Update Laravel framework
composer update laravel/framework
```

### 4.9 Troubleshooting Common Issues

**Issue 1: Halaman Blank/White Screen**

Possible causes:
- PHP error yang tidak ter-display
- Permission issue

Solution:
```bash
# Check permission
chmod -R 775 storage bootstrap/cache

# Enable debug (hanya untuk troubleshooting)
# Di .env: APP_DEBUG=true

# Check log
tail -f storage/logs/laravel.log
```

**Issue 2: Upload File Gagal**

Possible causes:
- File size melebihi upload_max_filesize
- Storage permission issue
- Disk space penuh

Solution:
```bash
# Check php.ini
upload_max_filesize = 100M
post_max_size = 100M

# Restart web server
sudo service apache2 restart
# atau
sudo service nginx restart

# Check disk space
df -h
```

**Issue 3: Database Connection Failed**

Solution:
```bash
# Check MySQL service
sudo service mysql status

# Restart MySQL
sudo service mysql restart

# Check credential di .env
# Test connection:
php artisan db:show
```

**Issue 4: Session Expired Terlalu Cepat**

Solution:
Edit `config/session.php`:
```php
'lifetime' => 120, // dalam menit
'expire_on_close' => false,
```

### 4.10 Performance Optimization

**Database Optimization:**

1. **Add Indexes:**
   ```sql
   -- Index pada tabel training assignments
   CREATE INDEX idx_user_id ON training_assignments(user_id);
   CREATE INDEX idx_training_master_id ON training_assignments(training_master_id);
   CREATE INDEX idx_status ON training_assignments(status);

   -- Index pada tabel exam results
   CREATE INDEX idx_exam_user ON exam_results(user_id, exam_id);
   ```

2. **Enable Query Cache:**
   Di `.env`:
   ```env
   DB_CACHE_ENABLED=true
   ```

**Caching Strategy:**

1. **Enable Redis Cache** (recommended for production):
   ```env
   CACHE_DRIVER=redis
   SESSION_DRIVER=redis
   QUEUE_CONNECTION=redis
   ```

2. **Configure Redis** di `config/database.php`:
   ```php
   'redis' => [
       'client' => env('REDIS_CLIENT', 'phpredis'),
       'default' => [
           'host' => env('REDIS_HOST', '127.0.0.1'),
           'password' => env('REDIS_PASSWORD', null),
           'port' => env('REDIS_PORT', 6379),
           'database' => env('REDIS_CACHE_DB', 0),
       ],
   ],
   ```

**Frontend Optimization:**

1. **Compile Assets untuk Production:**
   ```bash
   npm run build
   ```

2. **Enable HTTP Caching di Nginx:**
   ```nginx
   location ~* \.(css|js|png|jpg|jpeg|gif|ico)$ {
       expires 1y;
       add_header Cache-Control "public, immutable";
   }
   ```

## 5. TROUBLESHOOTING DAN FAQ

### 5.1 Troubleshooting untuk Peserta

**Problem: Tidak bisa login ke sistem**

| Possible Cause | Solution |
|----------------|----------|
| Username/password salah | Reset password melalui "Forgot Password" atau hubungi IT |
| Akun belum diaktifkan | Hubungi HR untuk aktivasi akun |
| Account locked | Tunggu 15 menit atau hubungi IT untuk unlock |
| Browser tidak kompatibel | Gunakan Chrome, Firefox, atau Edge versi terbaru |

**Problem: Training tidak muncul di daftar**

| Possible Cause | Solution |
|----------------|----------|
| Belum ditugaskan training | Hubungi HR untuk penugasan training |
| Training period belum dimulai | Tunggu hingga tanggal mulai |
| Training sudah expired | Hubungi HR untuk training baru |
| Filter salah | Clear filter dan refresh halaman |

**Problem: Materi tidak bisa dibuka**

| Possible Cause | Solution |
|----------------|----------|
| Koneksi internet tidak stabil | Cek koneksi internet |
| Browser cache penuh | Clear browser cache |
| File type tidak didukung | Update aplikasi pembaca file |
| Storage issue | Hubungi IT support |

**Problem: Ujian tidak bisa di-submit**

| Possible Cause | Solution |
|----------------|----------|
| Belum semua soal terjawab | Pastikan semua soal terjawab |
| Koneksi terputus saat submit | Coba kembali setelah koneksi stabil |
| Waktu habis | Sistem akan otomatis submit |
| Browser crash | Hubungi IT untuk cek status ujian |

---

### 5.2 Troubleshooting untuk Koordinator/HR

**Problem: Tidak bisa membuat training baru**

| Possible Cause | Solution |
|----------------|----------|
| Tidak memiliki permission | Hubungi IT untuk cek role dan permission |
| Required field belum diisi | Isi semua field wajib (required) |
| System error | Cek log dan hubungi IT support |

**Problem: Peserta tidak menerima notifikasi**

| Possible Cause | Solution |
|----------------|----------|
| Email configuration salah | Cek configuration mail di .env |
| Email masuk ke spam | Cek folder spam |
| Queue worker tidak berjalan | Jalankan queue worker: `php artisan queue:work` |
| Email service down | Cek status email service provider |

**Problem: Report tidak bisa di-export**

| Possible Cause | Solution |
|----------------|----------|
| Data terlalu besar | Export dengan filter lebih spesifik |
| Memory limit tercapai | Increase memory_limit di php.ini |
| Permission issue | Cek permission pada storage folder |

---

### 5.3 Troubleshooting untuk Pelaksana Training

**Problem: Tidak bisa membuka session**

| Possible Cause | Solution |
|----------------|----------|
| Training belum dimulai (sesuai tanggal) | Cek tanggal mulai training |
| Session sudah ada untuk hari ini | Tutup session yang lama terlebih dahulu |
| Tidak ditugaskan sebagai trainer | Hubungi HR untuk konfirmasi |

**Problem: Peserta tidak bisa check-in**

| Possible Cause | Solution |
|----------------|----------|
| Session code salah | Share ulang session code dengan benar |
| Session sudah ditutup | Buka session kembali |
| Peserta tidak terdaftar | Cek daftar peserta di sistem |

**Problem: Ujian tidak bisa dibuka**

| Possible Cause | Solution |
|----------------|----------|
| Materi belum selesai | Pastikan semua peserta selesai materi |
| Ujian tidak tersedia | Cek apakah bank soal sudah ditambahkan |
| Session belum dibuka | Buka session terlebih dahulu |

---

### 5.4 Troubleshooting untuk Admin IT

**Problem: Website lambat**

| Possible Cause | Solution |
|----------------|----------|
| Database query tidak optimal | Add indexes, optimize query |
| Cache tidak enabled | Enable config dan route cache |
| Server resources tidak cukup | Upgrade server atau scale horizontally |
| Too many concurrent users | Implement queue system, load balancing |

**Problem: Storage penuh**

| Possible Cause | Solution |
|----------------|----------|
| Terlalu banyak file upload | Implement cleanup old files scheduler |
| Log terlalu besar | Setup log rotation |
| Backup tidak ter-manage | Move backup ke external storage |

**Problem: Queue tidak berjalan**

Solution:
```bash
# Check queue status
php artisan queue:failed

# Restart queue worker
php artisan queue:restart

# Run queue manually
php artisan queue:work --tries=3 --timeout=90
```
