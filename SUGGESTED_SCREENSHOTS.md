## INSTRUKSI PENGAMBILAN SCREENSHOT

### Cara Mengambil Screenshot:
1. **Windows**: Tekan `Windows + Shift + S` untuk snipping tool, atau `Print Screen` untuk full screen
2. **Simpan** dengan nama yang jelas sesuai dengan nomor screenshot
3. **Format** yang disarankan: PNG atau JPG (kualitas baik, ukuran wajar)
4. **Ukuran yang disarankan**: Width 1200-1400px ( readable di berbagai device)
5. **Naming Convention**: `screenshot_[nomor]_[deskripsi].png`

### Tips Screenshot yang Baik:
- ✅ Tampilkan seluruh window atau halaman yang relevan
- ✅ Pastikan teks terbaca dengan jelas
- ✅ Hindari elemen yang tidak perlu (tab lain, notifikasi, dll)
- ✅ Zoom browser ke 100% untuk konsistensi
- ✅ Gunakan bahasa Indonesia di interface (jika ada opsi)
- ❌ Jangan include data pribadi/sensitif
- ❌ Jangan include error message atau debug info (kecuali sedang mendokumentasikan error)

## TIPS PENGORGANISASIAN FILE

### Struktur Folder yang Disarankan:

```
screenshots/
├── part-1-peserta/
│   ├── screenshot-01-login.png
│   ├── screenshot-02-dashboard.png
│   └── ...
├── part-2-koordinator/
│   ├── screenshot-14-hr-dashboard.png
│   └── ...
├── part-3-trainer/
│   ├── screenshot-41-trainer-dashboard.png
│   └── ...
└── part-4-admin-it/
    ├── screenshot-59-env-config.png
    └── ...
```

### Penamaan File:
- Gunakan nama deskriptif: `screenshot-[nomor]-[deskripsi-ringkas].png`
- Contoh: `screenshot-01-halaman-login.png`
- Hindari spasi, gunakan hyphen atau underscore

### Alternatif: Template Markdown untuk Insert Screenshot

Setelah screenshot diambil, Anda dapat mengganti placeholder di manual dengan:

**Format 1: Image Lokal**
```markdown
![Deskripsi Screenshot](../screenshots/part-1/screenshot-01-login.png)
```

**Format 2: Dengan Caption**
```markdown
<figure>
  <img src="../screenshots/part-1/screenshot-01-login.png" alt="Halaman Login">
  <figcaption>Gambar 1: Halaman login sistem training</figcaption>
</figure>
```

**Format 3: HTML (untuk styling lebih lanjut)**
```html
<div class="screenshot-container">
  <img src="../screenshots/part-1/screenshot-01-login.png" alt="Halaman Login">
  <p class="caption">Gambar 1: Halaman login sistem training</p>
</div>
```

   - Ambil screenshot dari environment development
   - Pastikan tampilan sama dengan production

3. **Hilangkan Placeholder Sama Sekali**
   - Hapus baris `**[SCREENSHOT X: ...]**` dari manual
   - Manual akan tetap useful tanpa gambar

4. **Rekaman Video**
   - Buat video tutorial sebagai alternatif screenshot
   - Upload ke YouTube atau platform internal
   - Link video di bagian yang relevan

## CATATAN PENTING

1. **Privasi dan Data Sensitif**
   - Jangan tampilkan data karyawan asli
   - Gunakan dummy data untuk screenshot
   - Mask atau blur informasi sensitif

2. **Konsistensi**
   - Gunakan browser dan ukuran window yang sama
   - Gunakan theme yang sama (jika ada dark/light mode)
   - Pastikan bahasa interface konsisten

3. **Kualitas**
   - Pastikan teks readable
   - Hindari screenshot yang terlalu kecil atau terlalu besar
   - Compress jika terlalu besar file size

4. **Update Berkala**
   - Screenshot perlu di-update jika UI berubah
   - Simpan source file (PSD, Figma, dll) untuk memudahkan update
